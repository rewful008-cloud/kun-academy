# دليل النشر - أكاديمية كن

## نشر المشروع على GitHub و Netlify

### 1. رفع المشروع على GitHub

#### الخطوات:
1. إنشاء مستودع جديد على GitHub باسم `kun-academy`
2. في مجلد المشروع، تشغيل الأوامر التالية:

```bash
git init
git add .
git commit -m "Initial commit: Kun Academy PWA"
git branch -M main
git remote add origin https://github.com/[username]/kun-academy.git
git push -u origin main
```

### 2. ربط المشروع بـ Netlify

#### الطريقة الأولى: ربط مستودع GitHub
1. تسجيل الدخول إلى [Netlify](https://netlify.com)
2. النقر على "New site from Git"
3. اختيار GitHub وتحديد مستودع `kun-academy`
4. إعدادات البناء:
   - **Build command:** (اتركه فارغاً)
   - **Publish directory:** `.` (النقطة تعني المجلد الجذر)
5. النقر على "Deploy site"

#### الطريقة الثانية: رفع مباشر
1. ضغط المجلد كاملاً في ملف ZIP
2. سحب وإفلات الملف على Netlify

### 3. تخصيص الدومين

#### تغيير اسم الموقع إلى kun-academy:
1. الذهاب إلى Site settings > General > Site details
2. النقر على "Change site name"
3. إدخال `kun-academy`
4. الدومين النهائي سيكون: `https://kun-academy.netlify.app/`

### 4. التحقق من إعدادات PWA

#### التأكد من الميزات التالية:
- ✅ Service Worker يعمل بشكل صحيح
- ✅ Manifest.json محمل ومُعرَّف
- ✅ الموقع قابل للتثبيت على الموبايل
- ✅ نظام الإشعارات يعمل
- ✅ التحديثات التلقائية مفعلة

### 5. اختبار التثبيت على الموبايل

#### على Android:
1. فتح الموقع في Chrome
2. النقر على قائمة المتصفح (⋮)
3. اختيار "Add to Home screen" أو "Install app"

#### على iOS:
1. فتح الموقع في Safari
2. النقر على زر المشاركة
3. اختيار "Add to Home Screen"

### 6. اختبار نظام الإشعارات

#### التحقق من:
1. طلب إذن الإشعارات عند أول زيارة
2. إشعار الترحيب بعد منح الإذن
3. إشعارات التحديث عند توفر نسخة جديدة
4. إمكانية التفاعل مع الإشعارات

### 7. مراقبة الأداء

#### أدوات المراقبة:
- **Netlify Analytics:** لمراقبة الزيارات
- **Lighthouse:** لتقييم الأداء والـ PWA
- **Chrome DevTools:** لمراقبة Service Worker

### 8. التحديثات المستقبلية

#### عملية التحديث:
1. تعديل الملفات محلياً
2. رفع التغييرات على GitHub:
```bash
git add .
git commit -m "Update: [وصف التحديث]"
git push origin main
```
3. Netlify سيقوم بالنشر التلقائي
4. المستخدمون سيتلقون إشعارات التحديث

### 9. إعدادات الأمان

#### الميزات المفعلة:
- Content Security Policy (CSP)
- X-Frame-Options
- X-Content-Type-Options
- Referrer Policy
- Permissions Policy للإشعارات

### 10. استكشاف الأخطاء

#### المشاكل الشائعة وحلولها:

**مشكلة:** Service Worker لا يعمل
**الحل:** التأكد من أن الموقع يعمل على HTTPS

**مشكلة:** الإشعارات لا تظهر
**الحل:** التأكد من منح إذن الإشعارات في المتصفح

**مشكلة:** التطبيق لا يظهر للتثبيت
**الحل:** التأكد من صحة ملف manifest.json

### 11. معلومات تقنية

#### الملفات المهمة:
- `manifest.json`: إعدادات PWA
- `sw.js`: Service Worker للتخزين المؤقت والإشعارات
- `netlify.toml`: إعدادات Netlify
- `_redirects`: إعادة التوجيه للـ SPA

#### إصدارات التخزين المؤقت:
- الإصدار الحالي: `v3.0`
- يتم تحديث الإصدار تلقائياً مع كل تحديث
